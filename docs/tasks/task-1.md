> „Machen Sie die Kamera/ das Device erstmal zum Sprechen."

## Beschreibung
Wählen Sie eine ML-Library Ihrer Wahl (z.B. MediaPipe, TensorFlow.js, hand-pose-detection) und machen Sie Körperdaten live im Browser sichtbar. Keine Abstraktion, keine eigene Library – nur Rohdaten auf dem Screen.

Beobachten Sie: Welche Daten kommen rein? Was ist zuverlässig, was ist Rauschen? Wie verhält sich die Performance? Dokumentieren Sie Ihre Beobachtungen direkt im Repo.

## Akzeptanzkriterien
- Eine laufende Browser-Demo zeigt Live-Körperdaten aus der Kamera (Koordinaten, Landmarks o.ä.) oder Bewegungsdaten einem anderen Device sichtbar auf dem Screen.
- Die gewählte ML-Library ist begründet dokumentiert – warum diese, was kann sie, was nicht?
- Eine kurze Einschätzung der Datenqualität ist im Repo festgehalten: Was ist stabil, was ist unzuverlässig?
- Der Code ist im Repo eingecheckt und lokal startbar.
